# Руководство по анализу F-15 Strike Eagle II

## Обзор
Это руководство описывает систему анализа для реверс-инжиниринга DOS-исполняемых файлов F-15 Strike Eagle II. Система предоставляет продвинутые инструменты для анализа, сравнения и понимания 16-битных DOS-исполняемых файлов на уровне инструкций.

## Основной движок анализа

### 1. Система VariantMap
**Расположение:** [`analysis.cpp:38-137`](mzretools/src/analysis.cpp:38)

Класс `VariantMap` управляет отношениями между вариантами инструкций для идентификации эквивалентных последовательностей инструкций в разных исполняемых файлах.

**Ключевые методы:**
- [`VariantMap::checkMatch()`](mzretools/src/analysis.cpp:55): Определяет, являются ли две последовательности инструкций вариантами
- [`VariantMap::find()`](mzretools/src/analysis.cpp:92): Ищет совпадающие варианты в корзинах
- [`VariantMap::loadFromStream()`](mzretools/src/analysis.cpp:118): Загружает сопоставления вариантов из конфигурационных файлов

**Формат конфигурации:**
```
инструкция1/инструкция2/инструкция3
```

### 2. Симуляция состояния CPU
**Расположение:** [`analysis.cpp:139-253`](mzretools/src/analysis.cpp:139)

Реализует упрощенный эмулятор CPU для отслеживания состояний регистров во время анализа.

**Ключевые функции:**
- [`applyMov()`](mzretools/src/analysis.cpp:139): Обрабатывает симуляцию инструкции MOV
- [`applyPush()`](mzretools/src/analysis.cpp:211): Симулирует операции PUSH
- [`applyPop()`](mzretools/src/analysis.cpp:231): Симулирует операции POP

**Возможности:**
- Отслеживает значения регистров при выполнении инструкций
- Обрабатывает немедленные значения, регистр-в-регистр и память-в-регистр операции
- Управляет операциями стека с отслеживанием push/pop
- Создает сопоставления сегментов для регистров DS, ES и SS

### 3. Система сравнения инструкций
**Расположение:** [`analysis.cpp:257-285`](mzretools/src/analysis.cpp:257)

Предоставляет детальное сравнение между инструкциями с несколькими уровнями совпадения.

**Типы совпадений:**
- `INS_MATCH_FULL`: Полное совпадение инструкций
- `INS_MATCH_DIFF`: Разное сопоставление, но эквивалентное
- `INS_MATCH_DIFFOP1`: Разный первый операнд
- `INS_MATCH_DIFFOP2`: Разный второй операнд
- `INS_MATCH_MISMATCH`: Нет совпадения

### 4. Движок обнаружения дубликатов
**Расположение:** [`analysis.cpp:1529-1617`](mzretools/src/analysis.cpp:1529)

Реализует продвинутое обнаружение дублирующихся подпрограмм с использованием сопоставления по сигнатурам.

**Процесс:**
1. Генерирует сигнатуры для эталонных подпрограмм
2. Ищет совпадающие подпрограммы в целевых исполняемых файлах
3. Обрабатывает обнаружение коллизий, когда подпрограммы совпадают с несколькими сигнатурами
4. Предоставляет детальную статистику по скорости обнаружения дубликатов

### 5. Поиск ссылок на данные
**Расположение:** [`analysis.cpp:1620-1699`](mzretools/src/analysis.cpp:1620)

Поиск перебором для потенциальных ссылок на переменные в сегментах данных исполняемых файлов.

**Функциональность:**
- Сканирует сегменты данных для значений, совпадающих с известными адресами переменных
- Предоставляет подсчет ссылок и сортировку
- Идентифицирует потенциальные ложные срабатывания на основе частоты ссылок

## Структуры данных

### Система адресации
**Расположение:** [`address.h`](mzretools/include/dos/address.h)

**Ключевые структуры:**
- `Address`: Представляет сегментированные адреса (сегмент:смещение)
- `Block`: Представляет диапазоны адресов для подпрограмм и регионов памяти
- `Segment`: Управляет сегментами памяти с проверкой границ

**Константы:**
- `MEMORY_SIZE = 1_MB`: Всего адресуемой памяти
- `SEGMENT_SIZE = 64_kB`: Размер каждого сегмента
- `SEGMENT_MASK = 0xFFFF`: Маска границ сегмента

### Управление подпрограммами
**Расположение:** [`routine.h`](mzretools/include/dos/routine.h)

**Ключевые структуры:**
- `Routine`: Содержит метаданные подпрограммы, включая экстенты, достижимые/недостижимые блоки
- `RoutineEntrypoint`: Представляет точки входа подпрограмм с обнаружением коллизий

**Возможности:**
- Обнаружение коллизий между подпрограммами
- Управление блоками для регионов памяти
- Расчет экстентов для границ подпрограмм

## Процесс верификации

### 1. Сравнение бинарных файлов
Процесс верификации использует инструмент `mzdiff` для сравнения реконструированных бинарных файлов с эталонами.

**Формат команды:**
```bash
mzdiff --codemap map/egame.map --target build/EGAME.EXE --pattern [558bec83ec??c746] --entry 0x10 bin/egame.exe
```

### 2. Анализ отладочного вывода
**Распространенные отладочные сообщения:**

- **"WARNING: Unable to find segment for offset 0x10000"**
  - Указывает, что инструмент анализа не может определить, какой сегмент содержит смещение 0x10000
  - Обычно происходит, когда смещение превышает границы сегмента

- **"Code map generation: 126 routines across 2 segments"**
  - Показывает успешную идентификацию подпрограмм в сегментах памяти
  - Указывает на частичный успех в анализе кода

- **"Comparison result: mismatch"**
  - Реконструированный бинарный файл не совпадает с эталоном
  - Указывает на различия в реализации или компиляции

### 3. Параметры верификации
- **routineSizeThresh**: Минимальный размер подпрограммы (15 инструкций)
- **routineDistanceThresh**: Порог расстояния для сопоставления подпрограмм (10% соотношение)
- **entryPoint**: Начальный адрес для анализа (0x10)

## Шаблоны использования

### Типичный поток анализа
1. Загрузите целевой исполняемый файл с использованием класса `Executable`
2. Сгенерируйте сигнатуры подпрограмм из эталонного исполняемого файла
3. Используйте `findDuplicates()` для идентификации совпадающих подпрограмм
4. Примените `findDataRefs()` для определения ссылок на переменные
5. Используйте симуляцию состояния CPU для отслеживания регистров во время анализа

### Конфигурационные файлы
- **Сопоставления вариантов**: Текстовые файлы с определениями вариантов, разделенными слэшем
- **Отладочный вывод**: Управляется макросом `OUTPUT_CONF(LOG_ANALYSIS)`
- **Пороги расстояния**: Настраиваемые для обнаружения дубликатов

## Распространенные проблемы и решения

### 1. Ошибки сопоставления сегментов
**Проблема:** "Unable to find segment for offset"
**Решение:** Проверьте границы сегментов и расчеты смещений

### 2. Ошибки сопоставления подпрограмм
**Проблема:** Подпрограммы не совпадают
**Решение:** Настройте пороги расстояния или сопоставления вариантов

### 3. Нарушения доступа к памяти
**Проблема:** Ошибки доступа к памяти
**Решение:** Проверьте разрешения сегментов и границы

### 4. Несоответствие бинарных файлов
**Проблема:** "Comparison result: mismatch"
**Решение:** 
- Проверьте параметры компиляции
- Убедитесь в правильности смещений
- Проверьте порядок байтов (endianness)

## Инструменты отладки
- Включите подробное логирование с `OUTPUT_CONF(LOG_ANALYSIS)`
- Используйте `findDataRefs()` для определения местоположений переменных
- Проверьте сигнатуры подпрограмм со статистикой `findDuplicates()`

## Структура файлов
- `analysis.cpp`: Реализация основного движка анализа
- `address.h`: Управление адресацией и сегментами

- `codemap.cpp`: Генерация карт кода и идентификация подпрограмм
- `security.h`: Утилиты безопасного доступа к памяти
